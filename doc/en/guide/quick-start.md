# Quick Start

## Quick Start (Development Mode)

```powershell
uv venv
uv pip install -e packages/core -e packages/cli -e packages/transcriptome -e packages/docking
.\.venv\Scripts\bioanalyse.exe plugins
```

## Running Tests in Docker

For Windows users or developers who don't want to install complex dependencies locally, we provide a convenient way to run tests in Docker containers. This environment automatically mounts local code, data directories (`data`), and output directories (`output`), so:

1. Tests can be run without rebuilding the image after modifying code.
2. Test code can directly access files in the `data` directory.
3. Reports or files generated by tests can be viewed directly in the local `output` directory.

**Windows (PowerShell):**

```powershell
.\run_tests_docker.ps1
```

**Linux / macOS:**

```bash
./run_tests_docker.sh
```

This will automatically build the test image and run all test cases.
